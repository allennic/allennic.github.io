在nginx配置naxsi

一、命令行复制naxsi_config中的naxsi_core.rules到nginx的etc配置目录

二、在nginx的etc配置目录新建一个naxsi.rules

代码如下

SecRulesEnabled;

DeniedUrl “/RequestDenied”;

CheckRule “$SQL >= 8” BLOCK;

CheckRule “$RFI >= 8” BLOCK;

CheckRule “$TRAVERSAL >= 4” BLOCK;

CheckRule “$EVADE >= 4” BLOCK;

CheckRule “$XSS >= 8” BLOCK;

三、

在http{}中设置include ../naxsi_core.rules;

在server{}中设置 include ../naxsi.rules;

后记

一、搭建环境真的是各种问题，刚开始login.php空白以为是nginx无法解析php（后面翻文档，无法解析是页面直接输出php代码），试写了phpinfo发现可以解析，然后google了下又说是mysql的问题，又写了个查库的页面发现可以连接mysql并且输出。。最后还是换了php5.6.25才一路顺风

二、最近翻了些文档，自己总结的防火墙有三种，分别为部署在项目本身的PHPIDS、部署在web服务器边缘的modsecurity和naxsi等、部署在主机边界的iptables策略。希望自己都能过上几遍。。。

三、类似modsecurity这种WAF其实也可用sqlmap中强大的tamper绕过，今天重拾sqlmap发现多了好多东西。。

bug

ci与axios使用会触发一个致命的bug，是php5.6引起的。
具体为ci中$this->input->post接收axios.post会取不到值，网上解决方法用

json_decode(trim(file_get_contents(‘php://input’)), true);

虽然取到值了，但是返回给前端的json却带有warning，无法用js取到返回的指定的值。

补上面留下的坑，可以把warning去掉233。。js可以抓取指定的值

首先：
打开php.ini文件

添加这个配置:always_populate_raw_post_data = -1

或者添加这个:enable_post_data_reading = On(官方推荐这个)
